# SETUP_CHECKLIST.md — "Graded" v1

Credentials and accounts Tyler needs to provide, in the order they're needed.
Claude Code will ask for these by item number as milestones require them —
never all at once, never earlier than needed.

## A — Needed starting at M0

**A1. Telegram API credentials**
- Go to https://my.telegram.org → "API development tools" (log in with Tyler's own phone
  number — this creates a personal API app, not a bot).
- Create an app (name/description can be anything, e.g. "Graded Tracker").
- Copy the `api_id` (number) and `api_hash` (string). These become `TELEGRAM_API_ID` /
  `TELEGRAM_API_HASH`.
- Note: `TELEGRAM_SESSION` is NOT created here — it's generated later (M4) via an
  interactive login script Claude Code will walk Tyler through, using this api_id/api_hash.

**A2. Anthropic API key**
- https://console.anthropic.com → API Keys → Create Key.
- Add billing / a card so the key can make requests. Expected cost per §16: under
  $5/mo at 300 msgs/day; hard-capped at $25/mo by the app itself.
- Becomes `ANTHROPIC_API_KEY`.

**A3. Neon Postgres database**
- https://neon.tech → sign up (free tier is fine for v1) → New Project.
- Copy the **pooled** connection string (for the app) AND the **direct/unpooled**
  connection string (required for pg-boss's LISTEN/NOTIFY — see spec §20.5).
- Becomes `DATABASE_URL` (pooled) and a second var for the direct one, TBD name at
  build time.

**A4. Railway account**
- https://railway.app → sign up, connect GitHub.
- Claude Code will need a GitHub repo to deploy from (see A5) and Railway project
  access to set environment variables and trigger deploys.

**A5. GitHub repository**
- A new empty repo (private is fine) for this project, e.g. `github.com/<you>/graded`.
- Claude Code will push the initial scaffold here; Railway deploys from it.

## B — Needed later, not blocking M0

**B1. Admin password** — Tyler picks a long random password for `/admin` login
(no need to send it to Claude Code in chat; Tyler can set it directly as a Railway
env var, or share it here if he prefers Claude Code to set it for him — his call).

**B2. `SESSION_SECRET`** — generated by Claude Code via `openssl rand -hex 32`, no
input needed from Tyler.

**B3. Custom domain (optional)** — if/when Tyler wants a real domain instead of the
Railway-provided one, he'll need registrar access to add a CNAME.

**B4. Uptime monitor** — a free account at something like UptimeRobot or Better
Uptime, pointed at `/api/health` once M0 is deployed. Not urgent — can wait until M5.

**B5. Reddit API app** — needed now, moved up ahead of schedule at Tyler's request
(2026-07-14) to get automated handle-tracking working sooner than the original M6 plan.
- Go to https://www.reddit.com/prefs/apps (log in with your own Reddit account).
- Click "create another app..." near the bottom.
  - name: anything, e.g. "Graded Tracker"
  - type: select **script** (not "web app" or "installed app")
  - description: can leave blank
  - about url: required by the form but not actually checked — any valid URL works,
    e.g. `https://github.com/Tyler-Patsy/Bet_Tracker`
  - redirect uri: `http://localhost:8080` (required field, but unused for script apps)
- Click "create app".
- Copy two values from the box that appears:
  - the string under "personal use script" (right under the app name) — this is the
    **client ID**
  - the "secret" field — this is the **client secret**
- Send both here. These become `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET`.
